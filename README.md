## 🚀 Getting Started

### Prerequisites

Before you begin, ensure you have the following installed:

- **Git** (v2.25 or higher)
- **Node.js** (v16 or higher) - Optional, for development tools
- **Modern web browser** (Chrome, Firefox, Safari, Edge)
- **Code editor** (VS Code recommended)

### Installation

1. **Clone the repository**

```bash
git clone https://github.com/register360/webburnstech.com.git
cd webburnstech.com
```

2. **Open in browser**

```bash
# For simple viewing
open webburns_tech.html

# Or use a local server (recommended)
python -m http.server 8000
# Then visit http://localhost:8000
```

3. **For development** (optional)

```bash
# Install dependencies (if using build tools)
npm install

# Start development server
npm start
```

### Quick Start

```bash
# Clone and open
git clone https://github.com/register360/webburnstech.com.git
cd webburnstech.com
open webburns_tech.html
```

## 🔧 Development

### Development Setup

1. **Fork the repository** on GitHub

2. **Clone your fork**
```bash
git clone https://github.com/YOUR_USERNAME/webburnstech.com.git
cd webburnstech.com
```

3. **Create a feature branch**
```bash
git checkout -b feature/your-feature-name
```

4. **Make your changes** and test thoroughly

5. **Commit with conventional commits**
```bash
git commit -m "feat(portfolio): add project filtering"
```

6. **Push to your fork**
```bash
git push origin feature/your-feature-name
```

7. **Create a Pull Request** on GitHub

### Code Style

We follow industry-standard coding practices:

- **HTML**: Semantic HTML5, proper accessibility attributes
- **CSS**: BEM methodology, mobile-first approach
- **JavaScript**: ES6+, clean code principles, JSDoc comments

See [Code Style Standards](CONTRIBUTING.md#code-style-standards) for detailed guidelines.

### Git Workflow

```bash
# Update your local main branch
git checkout main
git pull upstream main

# Create feature branch
git checkout -b feature/new-feature

# Make changes and commit
git add .
git commit -m "feat: add new feature"

# Push to your fork
git push origin feature/new-feature
```

## 🚀 Deployment

### GitHub Pages

```bash
# Push to main branch
git push origin main

# GitHub Pages will automatically deploy
# Visit: https://register360.github.io/webburnstech.com/
```

### Netlify

1. Connect your GitHub repository to Netlify
2. Configure build settings:
   - **Build command**: None (static site)
   - **Publish directory**: `/`
3. Deploy automatically on push to main

### Custom Domain

1. Add your domain in hosting provider settings
2. Update DNS records:
   ```
   Type: CNAME
   Name: www
   Value: your-deployment-url
   ```
3. Enable HTTPS/SSL certificate

### Reporting Issues

When reporting issues, please include:

- **Description** - Clear description of the problem
- **Steps to Reproduce** - How to recreate the issue

- **Actual Behavior** - What actually happens

- **Screenshots** - Visual evidence if applicable
